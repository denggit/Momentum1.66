# 阿里云东京2C2G服务器部署指南

## 概述

本文档详细介绍了在阿里云东京地区2核2G服务器上部署四号引擎v3.0的完整流程，包括环境配置、依赖安装、系统优化和监控设置。

## 1. 服务器选型与配置

### 1.1 服务器规格

| 配置项 | 推荐规格 | 说明 |
|--------|----------|------|
| **地域** | 东京 (ap-northeast-1) | 靠近OKX服务器，延迟低 |
| **实例类型** | ecs.g6.large | 2核4G，突发性能实例 |
| **操作系统** | Ubuntu 22.04 LTS | 稳定性好，社区支持完善 |
| **系统盘** | 40GB ESSD云盘 | 高性能存储 |
| **带宽** | 按使用流量计费，峰值10Mbps | 节省成本 |

### 1.2 成本估算

| 项目 | 月度费用 (约) |
|------|--------------|
| ECS实例 | $15-20 |
| 系统盘 | $2-3 |
| 流量费用 | $5-10 (取决于使用量) |
| **总计** | **$22-33** |

```bash
sudo ufw allow 80/tcp
```


### 5.2 CPU亲和性设置

```python
# CPU亲和性脚本
# !/usr/bin/env python3
# ~/scripts/setup_cpu_affinity.py

import psutil
import os


def setup_cpu_affinity():
   """设置CPU亲和性"""
   process = psutil.Process()

   # 主进程绑定核心0
   process.cpu_affinity([0])
   print(f"主进程 {process.pid} 绑定核心0成功")

   # 设置子进程继承CPU亲和性
   os.environ['NUMBA_CPU_AFFINITY'] = "1"
   print("Worker进程将绑定核心1")


if __name__ == "__main__":
   setup_cpu_affinity()
```

## 6. 服务管理

### 6.1 Systemd服务配置

```bash
# 1. 创建服务文件
sudo vim /etc/systemd/system/triplea-engine.service
```

```ini
[Unit]
Description=Triple-A HFT Engine
After=network.target
Requires=network.target

[Service]
Type=simple
User=triplea
Group=triplea
WorkingDirectory=/home/triplea/Momentum1.66
Environment="PATH=/home/triplea/venv/triplea/bin"
EnvironmentFile=/home/triplea/Momentum1.66/.env
ExecStart=/home/triplea/venv/triplea/bin/python engines/engine_4_triplea/orchestrator.py
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

# 安全限制
NoNewPrivileges=true
ProtectSystem=strict
ReadWritePaths=/home/triplea/Momentum1.66/logs
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```

### 6.2 服务管理命令


### 7.2 自定义监控指标

```python
# ~/Momentum1.66/deployment/monitoring/metrics_collector.py
import time
from prometheus_client import Gauge, Counter, start_http_server


class TripleAMetrics:
   """四号引擎监控指标"""

   def __init__(self):
      # Tick处理延迟
      self.tick_latency = Gauge(
         'triplea_tick_latency_ms',
         'Tick处理延迟（毫秒）',
         ['component']
      )

      # 内存使用
      self.memory_usage = Gauge(
         'triplea_memory_usage_mb',
         '内存使用量（MB）',
         ['component']
      )

      # 信号生成统计
      self.signals_generated = Counter(
         'triplea_signals_total',
         '生成的信号总数',
         ['signal_type']
      )

   def start_metrics_server(self, port=9091):
      """启动监控服务器"""
      start_http_server(port)
      print(f"监控服务器启动在端口 {port}")
```

## 8. 备份与恢复

### 8.1 备份策略

```bash
#!/bin/bash
# ~/scripts/backup_triplea.sh

BACKUP_DIR="/home/triplea/backups"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份配置文件
tar -czf $BACKUP_DIR/config_$DATE.tar.gz \
    deployment/science_vessel/config.yaml \
    .env \
    .numba_config.yaml

# 备份日志（保留最近7天）
find logs/ -name "*.log" -mtime -7 | \
    tar -czf $BACKUP_DIR/logs_$DATE.tar.gz -T -

# 清理旧备份（保留30天）
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

echo "备份完成: $BACKUP_DIR"
```

### 8.2 自动备份

## 9. 故障排查

### 9.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| **服务无法启动** | 依赖缺失、配置错误 | 检查日志，验证依赖 |
| **无信号生成** | 数据源异常、参数不当 | 检查WebSocket连接，调整参数 |
| **内存泄漏** | 缓冲区无限增长 | 检查缓冲区大小限制 |
| **高延迟** | CPU亲和性未生效 | 验证进程绑定状态 |

### 9.2 诊断命令

```bash
# 1. 检查服务状态
sudo systemctl status triplea-engine

# 2. 查看实时日志
sudo journalctl -u triplea-engine -f

# 3. 检查进程资源使用
top -p $(pgrep -f triplea)

# 4. 检查网络连接
netstat -an | grep 8443

# 5. 检查Numba缓存
ls -la ~/.cache/numba/

# 6. 检查磁盘空间
df -h
du -sh ~/Momentum1.66/logs/
```
