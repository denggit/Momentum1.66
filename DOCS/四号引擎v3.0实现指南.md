# 四号引擎v3.0实现指南

## 概述

本文档详细介绍了四号引擎v3.0纯Python高性能优化版的实现细节，包括架构设计、核心组件、数据流和性能优化策略。

## 1. 系统架构

### 1.1 双核隔离主从进程架构

四号引擎v3.0采用创新的双核隔离架构，将I/O密集型任务与CPU密集型任务严格分离：

- **主进程（核心0）**：处理WebSocket连接、状态机逻辑、风险管理和订单执行
- **从进程池（核心1）**：专用于KDE计算和LVN提取等CPU密集型任务

### 1.2 核心组件

| 组件 | 功能 | 实现位置 |
|------|------|----------|
| ProcessPoolManager | 进程池管理 | `src/strategy/triplea/process_pool_manager.py` |
| IPCProtocol | 进程间通信协议 | `src/strategy/triplea/ipc_protocol.py` |
| CPUAffinityManager | CPU亲和性绑定 | `src/strategy/triplea/cpu_affinity.py` |
| Serialization | 高性能序列化 | `src/strategy/triplea/serialization.py` |

## 4. 状态机设计

### 4.1 5状态模型

1. **IDLE**：空闲状态，等待市场数据
2. **MONITORING**：监控状态，检测潜在机会
3. **CONFIRMED**：确认状态，验证信号有效性
4. **ACCUMULATING**：积累状态，等待最佳入场时机
5. **POSITION**：持仓状态，管理止损止盈

### 4.2 状态转换条件

| 转换 | 触发条件 |
|------|----------|
| IDLE → MONITORING | 检测到潜在LVN区域 |
| MONITORING → CONFIRMED | CVD确认 + 成交量验证 |
| CONFIRMED → ACCUMULATING | 价格进入LVN区域 |
| ACCUMULATING → POSITION | 突破信号 + 风控检查通过 |
| POSITION → IDLE | 止损/止盈触发或手动平仓 |

## 8. 故障排除

### 8.1 常见问题

1. **Numba编译失败**：检查Python版本和Numba兼容性
2. **CPU亲和性设置失败**：检查Linux内核版本和权限
3. **内存泄漏**：使用`tracemalloc`进行内存分析
4. **WebSocket连接断开**：检查网络连接和OKX API状态

### 8.2 调试工具

- `tools/check_cpu_affinity.py`：CPU亲和性检查
- `tools/measure_process_isolation.py`：进程隔离性能测试
